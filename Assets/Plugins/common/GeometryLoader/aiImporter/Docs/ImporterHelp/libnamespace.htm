 DataLib Namespace

The DataLib library consists of 6 classes that hold the necessary model data. Eventhough the purpose of the aiImporter is to import a single skinned model, it can import multiple models if they are not skinned. That is why there is a GameScene class that contains a list of multiple models. Another purpose of having a Scene class is to create a similar structure to Assimp Library.


Class Name Description
GameScene Represents a completely imported model or scene. Every data that aiImporter has collected so far can be accesed here. This is the scene that we pass to Unity to create a model at runtime.
Model Represents a single model.
Geometry Represents geometry of a model with a single material. (Mesh data)
BoneData Represents a single bone of a mesh. Each bone stores its own name and its parent's name so that we can create the hierarchy easier.
WeightData Represents a single influence of a bone on a vertex.
MaterialData Represents a single material that was used in the scene. It contains texture and color properties. Materials are stored directly in a List in the GameScene class. Geometry class just has a MaterialIndex to get the required material from the list.
